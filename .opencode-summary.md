# M3U Export Plugin - Session Summary

## Goal
Fix plugin API compatibility for M3U Export plugin on Jellyfin 10.11 (SortOrder + GetItemList + GetUserById errors).

## Constraints & Preferences
- Plugin installed via jprm from GitHub repository
- User runs Jellyfin 10.11.x (Docker, linuxserver image)
- Config page must work with Jellyfin Dashboard via `viewContainer.js`
- API token auth, anonymous M3U endpoints

## Progress

### v1.0.0.0–v1.0.0.4
- Config page fixes, checksum issues resolved
- Unusable on Jellyfin 10.11 due to `MissingMethodException: GetUserById(Guid)`

### v1.0.0.5 — Fixed GetUserById
- Removed `IUserManager`, `GetUserById()`, `ResolveUser()` calls
- Replaced with `ValidateToken()` for API key auth
- Queries use `new InternalItemsQuery { }` without userId filtering
- Result: loaded but `TypeLoadException: SortOrder` from Jellyfin.Data.Enums

### v1.0.0.6 — Fixed SortOrder TypeLoadException
- Removed all `OrderBy` with `SortOrder` from ExportMovies, ExportSeries, ExportCollections
- Build succeeded, installed, loaded
- Result: `MissingMethodException: GetItemList(InternalItemsQuery)` — return type changed from `List<BaseItem>` to `IReadOnlyList<BaseItem>` in Jellyfin 10.11

### v1.0.0.7 — Fixed GetItemList MissingMethodException
- Changed `_libraryManager.GetItemList(query)` → `_libraryManager.GetItemsResult(query).Items`
- `GetItemsResult` returns `QueryResult<BaseItem>` (same signature in 10.10 and 10.11)
- `QueryResult.Items` is `IReadOnlyList<BaseItem>` in both versions — no breakage
- **SHA256**: `d9429149f29cdc8fa75766299e48ac822c83c109d26b97898a77ba0f3490e43e`

## Key Decisions
- Removed `SortOrder` entirely (vs chasing moved namespace) — non-essential for M3U
- Removed `OrderBy` from all queries — simpler than adding refs to 10.11-only assemblies
- Used `GetItemsResult` instead of `GetItemList` — same semantics, stable signature
- All versions target ABI `10.10.0.0` — compiled against NuGet 10.10, compatible by contract with 10.11
